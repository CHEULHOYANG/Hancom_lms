# MND LMS 시스템 요구사항 명세서

## ? 프로젝트 개요

### 1.1 프로젝트 명
- **프로젝트명**: 국방부 학습관리시스템 (MND LMS)
- **영문명**: Ministry of National Defense Learning Management System
- **버전**: v1.0
- **개발기간**: 2025.11

### 1.2 목적 및 배경
국방부 교육훈련 체계의 디지털 전환을 통해 효율적이고 체계적인 학습 환경을 제공하고, 개인 맞춤형 교육과정 관리를 통한 교육 품질 향상을 목적으로 함.

## ? 기능 요구사항

### 2.1 사용자 관리 (User Management)
- **UR-001**: 사용자 회원가입 및 로그인 기능
- **UR-002**: 국방부 특화 사용자 정보 관리 (계급, 등급, 부대 등)
- **UR-003**: 개인정보 마스킹 처리 기능
- **UR-004**: 사용자 권한 관리 (관리자/일반사용자)
- **UR-005**: 사용자 활동 로그 관리

### 2.2 강의 관리 (Course Management)
- **CR-001**: 강의 카테고리별 분류 관리
- **CR-002**: 강의 등록, 수정, 삭제 기능
- **CR-003**: 강의 상태 관리 (게시됨/비공개/준비중)
- **CR-004**: 수강 정원 관리 및 대기열 시스템
- **CR-005**: 강의 진도율 추적 기능

### 2.3 수강 관리 (Enrollment Management)
- **ER-001**: 수강신청 및 취소 기능
- **ER-002**: 수강 이력 관리
- **ER-003**: 학습 진도 추적
- **ER-004**: 수료증 발급 기능
- **ER-005**: 평가 및 성적 관리

### 2.4 커뮤니티 (Community)
- **CM-001**: 게시판 시스템 (공지사항, 자유토론, Q&A)
- **CM-002**: 게시글 작성, 수정, 삭제 기능
- **CM-003**: 댓글 및 답글 기능
- **CM-004**: 파일 첨부 기능
- **CM-005**: 검색 기능

### 2.5 관리자 기능 (Admin Functions)
- **AD-001**: 관리자 대시보드
- **AD-002**: 사용자 관리 (조회, 수정, 삭제)
- **AD-003**: 강의 관리 (등록, 수정, 삭제)
- **AD-004**: 게시판 관리
- **AD-005**: 사이트 설정 관리
- **AD-006**: 통계 및 리포트 기능

## ? 비기능 요구사항

### 3.1 성능 요구사항
- **PR-001**: 웹페이지 로딩 시간 3초 이내
- **PR-002**: 동시 접속자 500명 지원
- **PR-003**: 데이터베이스 응답시간 1초 이내
- **PR-004**: 파일 업로드 최대 100MB 지원

### 3.2 보안 요구사항
- **SR-001**: HTTPS 통신 암호화
- **SR-002**: 사용자 인증 및 권한 관리
- **SR-003**: SQL 인젝션 방지
- **SR-004**: XSS(Cross-Site Scripting) 방지
- **SR-005**: CSRF(Cross-Site Request Forgery) 방지
- **SR-006**: 개인정보 마스킹 처리
- **SR-007**: 세션 보안 관리

### 3.3 가용성 요구사항
- **AR-001**: 시스템 가동율 99% 이상
- **AR-002**: 장애 복구 시간 1시간 이내
- **AR-003**: 정기 백업 (일일, 주간)
- **AR-004**: 24시간 모니터링 체계

### 3.4 호환성 요구사항
- **CR-001**: 웹 브라우저 호환성 (Chrome, Firefox, Safari, Edge)
- **CR-002**: 반응형 웹 지원 (PC, 태블릿, 모바일)
- **CR-003**: 다국어 지원 대비 (한국어 우선)

## ? 사용자 인터페이스 요구사항

### 4.1 디자인 요구사항
- **UI-001**: 국방부 CI/CD 가이드라인 준수
- **UI-002**: 직관적이고 사용하기 쉬운 인터페이스
- **UI-003**: 웹 접근성 가이드라인 준수
- **UI-004**: 반응형 디자인 적용

### 4.2 네비게이션 요구사항
- **NV-001**: 명확한 메뉴 계층 구조
- **NV-002**: 현재 위치 표시 (Breadcrumb)
- **NV-003**: 검색 기능 제공
- **NV-004**: 모바일 친화적 네비게이션

## ?? 데이터 요구사항

### 5.1 데이터 모델
- **CustomUser**: 사용자 정보 및 국방부 특화 필드
- **Course**: 강의 정보
- **CourseCategory**: 강의 카테고리
- **Enrollment**: 수강신청 관리
- **Board**: 게시판 정보
- **Post**: 게시글 관리
- **SiteSettings**: 사이트 설정

### 5.2 데이터 보안
- **DS-001**: 개인정보 암호화 저장
- **DS-002**: 데이터 접근 로그 관리
- **DS-003**: 정기적인 데이터 백업
- **DS-004**: 데이터 무결성 검증

## ? 시스템 환경 요구사항

### 6.1 개발 환경
- **Python**: 3.11+
- **Django**: 5.2.8
- **Database**: SQLite (개발), PostgreSQL (운영)
- **Web Server**: Gunicorn + Nginx
- **OS**: Ubuntu 22.04 LTS

### 6.2 운영 환경
- **클라우드**: Render Platform
- **CDN**: Cloudflare (권장)
- **모니터링**: Sentry (오류 추적)
- **로그**: Django Logging + 외부 로그 서비스
